# 版本与运行环境

## 前置条件
- 你需要确认当前接入的是哪个 `ocorm` 版本
- 你知道项目是否由 2.x 升级到 3.x

## 当前文档对齐范围
- 文档目标版本：`3.0.2`
- 包名：`ocorm`
- 运行环境：HarmonyOS / OpenHarmony ArkTS（依赖 `@kit.ArkData`、`@kit.AbilityKit`）

## 兼容性要点（基于 CHANGELOG 与源码）
- 3.x 保持 2.x 主调用方式：`import { ... } from 'ocorm'`。
- 3.x 中 `DatabaseConfig.encrypt` 默认值为 `true`，从 2.x 升级时应显式确认是否符合你的发布策略。
- 实体模式查询明确限制：`selectRaw/groupBy/having` 不可直接配合 `get()/count()`，应使用 `getRaw()` 或 `aggregate()`。

## 可复制的版本兼容初始化代码
```ts
import { Context } from '@kit.AbilityKit'
import { DatabaseConfig, OCORMInit } from 'ocorm'

export async function initWithExplicitCompatibility(context: Context): Promise<void> {
  // 显式传入 encrypt，避免 2.x -> 3.x 迁移时默认值歧义
  const config = new DatabaseConfig('compat.db', undefined, true)
    .setQueryTimeout(1500)
    .setRelationInClauseLimit(500)

  await OCORMInit(context, {
    config,
    autoCreateTables: false
  })
}
```

## 验证步骤
1. 执行 `await initWithExplicitCompatibility(this.context)`。
2. 无异常即表示当前运行环境可完成数据库初始化。
3. 升级项目再补做一次最小闭环（见 [03-10分钟最小闭环](./03-10分钟最小闭环.md)）。

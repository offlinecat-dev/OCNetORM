# 安装与导入约定

当前包名统一为 `ocorm`（见仓库 `oh-package.json5`）。

## 前置条件
- 已安装并可使用 `ohpm`
- 当前工程可以修改 `oh-package.json5`
- 你准备使用公开入口 `Index.ets` 的导出 API（不是内部路径）

## 安装
```bash
ohpm i @offlinecat/ocorm
```

`oh-package.json5` 建议固定到已验证版本：
```json5
{
  "dependencies": {
    "ocorm": "3.0.2"
  }
}
```

## 导入约定（强制）
- 统一使用 `from 'ocorm'`
- 不要使用 `from 'ocorm/src/main/ets/...'` 这类内部路径
- 不要复制仓库源码文件到业务工程再手工维护

## 可复制的安装后导入自检
```ts
import { Context } from '@kit.AbilityKit'
import {
  ColumnType,
  DatabaseConfig,
  EntityData,
  EntityDataInput,
  OCORMInit,
  Repository,
  defineEntity
} from 'ocorm'

let importedReady = false

export async function verifyInstallAndImport(context: Context): Promise<void> {
  if (!importedReady) {
    defineEntity('ImportCheckUser', {
      tableName: 'import_check_users',
      columns: [
        { property: 'id', name: 'id', primaryKey: true, autoIncrement: true },
        { property: 'name', name: 'name', type: ColumnType.TEXT }
      ]
    })

    await OCORMInit(context, {
      config: new DatabaseConfig('import-check.db'),
      autoCreateTables: true
    })

    importedReady = true
  }

  const repo = new Repository('ImportCheckUser')
  const input = EntityDataInput.create()
  input.set('name', 'import-ok')
  await repo.save(EntityData.from('ImportCheckUser', input))
}
```

## 验证步骤
1. 执行 `ohpm i @offlinecat/ocorm`。
2. 执行 `await verifyInstallAndImport(this.context)`。
3. 无异常即表示“安装 + 导入 + 基础写入”链路正常。

## 常见错误
- `数据库未初始化，请先调用 initialize() 方法`
  - 原因：未先调用 `OCORMInit(...)` 就开始查询或写入。
- `实体 "Xxx" 未注册`
  - 原因：`new Repository('Xxx')` 前没有 `defineEntity('Xxx', ...)`。
- `Cannot find module 'ocorm'`（编译期）
  - 原因：依赖未安装或 `oh-package.json5` 未生效。

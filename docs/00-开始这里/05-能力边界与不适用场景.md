# 能力边界与不适用场景

## 前置条件
- 你已确认当前项目是本地 SQLite 场景
- 你接受通过公开 API（`Index.ets` 导出）接入，而非依赖内部实现文件

## 适用场景
- 本地 SQLite 单机数据访问（`DatabaseConfig` + `Repository` + `QueryExecutor`）
- 需要实体建模、查询构建、事务、迁移、验证与钩子的一体化方案
- 需要较强“错误可定位性”（错误类与错误码已统一导出）

## 不适用场景
- 需要跨进程或跨服务的分布式事务
- 需要远程 MySQL/PostgreSQL 等驱动直连
- 依赖内部实现细节且不愿随版本升级调整

## 场景边界自检代码
```ts
import { Context } from '@kit.AbilityKit'
import { ColumnType, DatabaseConfig, EntityData, EntityDataInput, OCORMInit, Repository, defineEntity } from 'ocorm'

let boundaryReady = false

export function assertOcormScenario(remoteDriver: string = '', needDistributedTx: boolean = false): void {
  if (remoteDriver.length > 0) {
    throw new Error('ocorm 当前仅支持本地 SQLite，不支持远程数据库驱动')
  }
  if (needDistributedTx) {
    throw new Error('ocorm 当前不提供跨进程分布式事务能力')
  }
}

export async function runLocalBoundaryDemo(context: Context): Promise<void> {
  assertOcormScenario('', false)

  if (!boundaryReady) {
    defineEntity('BoundaryUser', {
      tableName: 'boundary_users',
      columns: [
        { property: 'id', name: 'id', primaryKey: true, autoIncrement: true },
        { property: 'name', name: 'name', type: ColumnType.TEXT }
      ]
    })

    await OCORMInit(context, {
      config: new DatabaseConfig('boundary.db'),
      autoCreateTables: true
    })

    boundaryReady = true
  }

  const repo = new Repository('BoundaryUser')
  await repo.transaction(async (txRepo) => {
    const input = EntityDataInput.create()
    input.set('name', 'local-only')
    await txRepo.save(EntityData.from('BoundaryUser', input))
  })
}
```

## 验证步骤
1. 执行 `await runLocalBoundaryDemo(this.context)`，确认本地事务链路可用。
2. 执行 `assertOcormScenario('mysql', false)`，应得到“仅支持本地 SQLite”错误。
3. 执行 `assertOcormScenario('', true)`，应得到“不提供分布式事务”错误。

## 常见错误（边界相关）
- `实体 "Xxx" 未注册`
  - 场景：直接 `new Repository('Xxx')`，但未 `defineEntity('Xxx', ...)`。
- `数据库未初始化，请先调用 initialize() 方法`
  - 场景：未执行 `OCORMInit(...)` 或连接已关闭后直接查询。
- `同一 QueryExecutor 实例不支持并发复用，请为并发操作创建独立实例`
  - 场景：同一个执行器对象被并发 `get()/count()/getRaw()` 复用。

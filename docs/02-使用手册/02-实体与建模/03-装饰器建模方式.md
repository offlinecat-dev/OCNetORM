# 装饰器建模方式

## 何时用
团队偏好类声明式模型，且希望注解表达实体约束时。

## 怎么用
使用 `Table/Column/PrimaryKey`，并保证注册顺序正确：先实体、后列。

```ts
import { Column, ColumnType, MetadataStorage, PrimaryKey, Table, UpdatedAt } from 'ocorm'

MetadataStorage.resetInstance()

class UserEntity {}
const user = new UserEntity()

// 1) 先注册实体
Table('users')(user.constructor as Function)

// 2) 再注册列
PrimaryKey({ name: 'id', autoIncrement: true })(user, 'id')
Column({ name: 'user_name', type: ColumnType.TEXT, nullable: false, unique: true })(user, 'userName')
UpdatedAt()(user, 'updatedAt')
```

## 最小可运行模板
```ts
import { Column, ColumnType, CreatedAt, PrimaryKey, Table } from 'ocorm'

class TodoEntity {}
const todo = new TodoEntity()

Table('todos')(todo.constructor as Function)
PrimaryKey({ autoIncrement: true })(todo, 'id')
Column({ type: ColumnType.TEXT, nullable: false })(todo, 'title')
Column({ type: ColumnType.INTEGER, defaultValue: 0 })(todo, 'done')
CreatedAt()(todo, 'createdAt')
```

## 失败示例与修复
失败示例（列先注册）：
```ts
class BrokenTodo {}
const todo = new BrokenTodo()
Column({ type: ColumnType.TEXT })(todo, 'title') // EntityNotRegisteredError
```

修复：
```ts
Table('todos')(todo.constructor as Function)
Column({ type: ColumnType.TEXT })(todo, 'title')
```

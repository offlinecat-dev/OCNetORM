# 术语表

## 何时用

在阅读使用手册前统一 API 语义，避免把“公开能力”和“内部实现细节”混用。

## 怎么用

- 只使用本表中的公开术语做沟通。
- “源码定义位置”指主定义文件（非测试文件）。
- 每个术语都先看“何时使用/何时不要用”，再进入对应章节。

## 常见误用

- 把公开术语当“某个内部文件名”来理解。
- 在没有初始化/注册实体的情况下直接调用仓储或查询 API。
- 把缓存、类型转换、校验当作数据库约束的替代品。

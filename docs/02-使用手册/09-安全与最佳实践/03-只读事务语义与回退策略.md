# 只读事务语义与回退策略

只读事务用于保护查询流程，避免误写。

## 建议
- 查询场景优先使用只读事务
- 若底层不支持只读模式，按业务要求决定 fail-fast 或回退
- 不要在只读事务中混入写操作

## 示例
```ts
import { Repository, TransactionOptions } from 'ocorm'

const repo = new Repository('User')
await repo.transactionWithOptions(async (txRepo) => {
  // 查询逻辑
}, TransactionOptions.readOnly())
```

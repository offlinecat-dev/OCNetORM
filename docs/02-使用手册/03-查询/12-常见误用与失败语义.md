# 常见误用与失败语义

> 源码对齐：`src/main/ets/query/QueryExecutor.ets`、`src/main/ets/database/manager/QuerySlotGate.ets`

## 失败语义速查表

| 场景 | 触发点 | 实际异常 | 关键消息片段 | 修复建议 |
|---|---|---|---|---|
| 同一 `QueryExecutor` 并发复用 | `withStoreLease` | `ExecutionError('QUERY_EXECUTOR', ...)` | `同一 QueryExecutor 实例不支持并发复用` | 并发任务必须创建独立 `QueryExecutor` |
| 查询并发槽位耗尽并超时 | `DatabaseManager.acquireQuerySlot()` | 外层包装为 `ExecutionError('SELECT'/'COUNT'/...)` | `等待查询槽位超时` | 提升 `maxConcurrentQueries` 或缩短单次查询持有时间 |
| 实体模式误用 raw 聚合 | `ensureEntityQueryMode` | `ExecutionError('SELECT'/'COUNT', ...)` | `实体查询不支持 selectRaw/groupBy/having` | 改用 `getRaw()` 或 `aggregate()` |
| `getRaw` + whereExists 内存过滤 | `getRaw` | `ExecutionError('RAW_SELECT', ...)` | `getRaw 不支持 whereExists 的内存过滤` | 简化子查询或改实体模式 |
| `getRaw` 分批子查询但缺少排序列 | `getRaw` | `ExecutionError('RAW_SELECT', ...)` | `要求 SELECT 显式包含 ORDER BY 列` | 把 `orderBy` 列加入 `select/selectRaw` |

## 可运行示例（并发正确姿势）

```arkts
import { Context } from '@kit.AbilityKit'
import {
  ColumnType,
  DatabaseConfig,
  OCORMInit,
  QueryBuilder,
  QueryExecutor,
  defineEntity
} from 'ocorm'

async function bootstrap(context: Context): Promise<void> {
  defineEntity('User', {
    tableName: 'users',
    columns: [
      { property: 'id', name: 'id', type: ColumnType.INTEGER, primaryKey: true, autoIncrement: false },
      { property: 'name', name: 'user_name', type: ColumnType.TEXT }
    ]
  })

  await OCORMInit(context, {
    config: new DatabaseConfig('query_failure_semantics.db')
      .setMaxConcurrentQueries(1)
      .setQueryTimeout(200),
    autoCreateTables: true
  })
}

export async function runParallelQuerySafely(context: Context): Promise<void> {
  await bootstrap(context)
  const qb = new QueryBuilder('User')

  // 正确做法：并发场景使用两个独立执行器实例
  const [rows, total] = await Promise.all([
    new QueryExecutor(qb.clone()).get(),
    new QueryExecutor(qb.clone()).count()
  ])

  console.info(rows.length, total)
}
```

## 失败案例 A：复用同一执行器并发调用

```arkts
import { Context } from '@kit.AbilityKit'
import { QueryBuilder, QueryExecutor } from 'ocorm'

export async function failReuseExecutorConcurrently(context: Context): Promise<void> {
  await bootstrap(context)
  const executor = new QueryExecutor(new QueryBuilder('User'))
  try {
    await Promise.all([executor.get(), executor.count()])
  } catch (e) {
    // 源码语义：ExecutionError('QUERY_EXECUTOR', ...)
    console.error('expected:', e)
  }
}
```

## 失败案例 B：并发槽位超时

```arkts
import { Context } from '@kit.AbilityKit'
import { DatabaseManager, QueryBuilder, QueryExecutor } from 'ocorm'

export async function failOnQuerySlotTimeout(context: Context): Promise<void> {
  await bootstrap(context)
  // bootstrap 已设置: maxConcurrentQueries = 1, queryTimeoutMs = 200
  await DatabaseManager.getInstance().acquireQuerySlot()
  try {
    await new QueryExecutor(new QueryBuilder('User')).get()
  } catch (e) {
    // 源码语义：ExecutionError('SELECT', '等待查询槽位超时 (100ms)')
    console.error('expected slot timeout:', e)
  } finally {
    DatabaseManager.getInstance().releaseQuerySlot()
  }
}
```

## 排障顺序（建议固定）
1. 先确认是否错误复用了同一 `QueryExecutor`。
2. 再看 `maxConcurrentQueries/queryTimeoutMs` 是否导致槽位排队超时。
3. 最后确认实体模式与原生模式是否混用。

# 07-查询异常排查

> 状态：已完成
> 适用范围：实体查询、原生查询、关系加载、超时与结果集异常
> 最后更新：2026-03-28

## 1. 高频异常类型
当前仓库里，查询类异常通常集中在四类：
- 实体模式误用聚合语义
- 原生 SQL 参数化/只读校验失败
- 关系加载被超时或边界限制跳过
- `ResultSet` 迟到关闭或连接已关闭导致的后续异常

## 2. 实体模式误用聚合语义
如果把 `selectRaw/groupBy/having` 和实体模式 `get()/getOne()/count()` 混在一起，当前实现会直接拒绝。

```arkts
await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['COUNT(*) AS total'])
).get()
```

正确做法是改走 `getRaw()` 或 `aggregate()`。

```arkts
await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['COUNT(*) AS total'])
).getRaw()
```

## 3. 原生查询校验失败
`rawQuery/rawQuerySafe` 当前必须参数化，而且只能执行只读 SQL。

```arkts
await repo.rawQuery('SELECT * FROM user')
await repo.rawQuerySafe('DELETE FROM user WHERE id = ?', [1])
```

排查关键词：

```text
RAW_QUERY 仅允许参数化 SQL
RAW_QUERY 参数数量与占位符数量不一致
RAW_QUERY_SAFE 仅允许参数化 SQL
RAW_QUERY_SAFE 参数数量与占位符数量不一致
RAW_QUERY 仅允许 SELECT/WITH/EXPLAIN 只读 SQL
RAW_QUERY 禁止写操作 SQL
```

## 4. 关系加载超时或分批行为异常
关系加载不是无限并发，也不是无限长 `IN (...)`。当前实现受 `relationInClauseLimit` 和加载超时控制。

```arkts
import { DatabaseConfig } from 'ocorm'

const config = new DatabaseConfig('app.db')
  .setRelationInClauseLimit(200)
```

当你发现“一部分关系没加载”或“批量关系查询 SQL 次数异常”，优先检查：
- `DatabaseConfig.relationInClauseLimit`
- `RelationLoader.setInClauseLimit(...)`
- 对应源码：`RelationLoader.ets`、`QueryExecutorRelationSupport.ets`

## 5. `ResultSet` / 连接关闭类问题
如果报错里出现：

```text
The RdbStore or ResultSet is already closed.
```

要先区分是：
- 测试上下文提早关闭了 store
- 查询超时后迟到的 `ResultSet` 被关闭
- 关系加载或原生查询在错误连接上继续执行

复现实验可参考当前仓库里对迟到结果集的测试写法。

```arkts
import { QueryBuilder, QueryExecutor } from 'ocorm'

const builder = new QueryBuilder('User').timeout(20)
QueryExecutor.setResultSetDelayMsForTest(80)
await new QueryExecutor(builder).get()
```

## 6. 排障顺序
- 先判断是实体查询、原生查询、关系加载还是测试清理问题
- 再检查有没有超时、连接关闭、参数化失败、语义混用
- 如果是关系加载，确认 `relationInClauseLimit` 和 timeout 是否过低
- 如果是原生查询，确认 SQL 是否参数化、是否只读

```text
建议顺序:
1. 错误消息里是否有 RAW_QUERY / RAW_EXECUTE / query_only / ResultSet closed
2. 是否使用了 selectRaw/groupBy/having + get/getOne/count
3. 是否设置了 timeout，且值过小
4. 是否命中了 relationInClauseLimit 分批
5. 是否测试环境先 rollback/close 再继续访问查询结果
```

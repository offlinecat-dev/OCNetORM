# 02-版本行为变化-3.0.1-3.0.2

> 状态：已完成
> 适用版本：`3.0.1 -> 3.0.2`
> 最后更新：2026-03-28

## 1. 结论

`3.0.2` 对纯库用户最关键的行为变化只有一条主线：`rawQuery` 从“只读校验”收紧为“默认强制参数化 + 占位符数量严格匹配”。

另外一条迁移线主要影响测试代码：`DatabaseManager` 的测试桥接接口迁到 `DatabaseManagerTestHooks`（仍由包入口导出）。

## 2. 差异对照

| 主题 | 3.0.1 | 3.0.2 | 迁移动作 |
| --- | --- | --- | --- |
| `rawQuery` | 只读校验为主 | 默认强参数化（无 `?` 或参数数不匹配直接拒绝） | 全量改成占位符 SQL |
| 原生 SQL 守卫实现 | 分散在 `Repository` 逻辑里 | 拆分到 `rawsql/RawSqlGuards` | 业务代码无需引用内部模块 |
| `DatabaseManager` 测试桥接 | 旧命名路径仍被历史代码使用 | 推荐统一 `DatabaseManagerTestHooks` | 测试代码替换旧入口 |

## 3. 旧新写法对照

### 3.1 `rawQuery`（必须改）

```arkts
// 3.0.1 历史风险写法
const rows = await repo.rawQuery('SELECT id, name FROM user')
```

```arkts
// 3.0.2 对齐写法
const rows = await repo.rawQuery(
  'SELECT id, name FROM user WHERE is_active = ?',
  [1]
)
```

### 3.2 测试桥接（建议统一）

```arkts
// 旧入口（历史测试常见）
DatabaseManager.resetInstanceForTesting()
manager.applyRuntimeConfigForTesting(config)
```

```arkts
// 3.0.2 推荐入口
import { DatabaseManager, DatabaseManagerTestHooks } from 'ocorm'

const manager = DatabaseManager.getInstance()
DatabaseManagerTestHooks.applyRuntimeConfig(manager, config)
DatabaseManagerTestHooks.resetInstance()
```

## 4. 批量改造清单

### 4.1 查找 `rawQuery` 非参数化调用

```bash
rg -n "rawQuery\(" src test
```

逐条检查：

- SQL 是否包含 `?`
- `?` 数量是否与参数数组长度一致
- 是否仍存在拼接式 SQL

### 4.2 查找旧测试桥接入口

```bash
rg -n "getTestState|setTestState|applyRuntimeConfigForTest|resetInstanceForTesting" src test
```

若是业务代码，不应依赖测试桥接；若是测试代码，统一收敛到 `DatabaseManagerTestHooks`。

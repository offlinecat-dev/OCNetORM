# 01-升级指南-2.x到3.x

> 状态：已完成
> 适用版本：`2.x -> 3.0.2`
> 最后更新：2026-03-28

## 1. 迁移结论（直接可执行）

`3.x` 的主入口 API 仍是 `import { ... } from 'ocorm'`，但默认行为与安全语义明显收紧。真正导致升级失败的，通常不是“类找不到”，而是：

- 默认配置变化（`DatabaseConfig.encrypt = true`）
- 事务语义收紧（`serializable()` 直接 fail-fast，`REPEATABLE_READ/SERIALIZABLE` 执行期拒绝）
- 查询语义分层（实体查询路径拒绝 `selectRaw/groupBy/having`）
- 原生 SQL 强参数化（`rawQuery/rawQuerySafe/rawExecute` 都要求占位符数量匹配）

## 2. 必改项矩阵

| 主题 | 2.x 风险写法（典型） | 3.x 必须动作 | 源码锚点 |
| --- | --- | --- | --- |
| 数据库初始化 | 依赖 `encrypt` 默认值 | 显式声明 `encrypt` | `src/main/ets/database/DatabaseConfig.ets` |
| 聚合查询 | `selectRaw` 后继续 `get()/count()` | 改为 `getRaw()` 或 `aggregate()` | `src/main/ets/query/QueryExecutor.ets` |
| 事务隔离 | 继续使用 `serializable()/REPEATABLE_READ` | 改为 `READ_COMMITTED/READ_UNCOMMITTED` | `src/main/ets/repository/TransactionOptions.ets`、`TransactionManager.ets` |
| 只读事务 | 假设 `readOnly` 自动降级 | 捕获 fail-fast（`PRAGMA query_only` 不支持会失败） | `src/main/ets/repository/TransactionManager.ets` |
| 原生 SQL | 字符串拼接 SQL / 参数数不匹配 | 全量参数化 + 占位符严格匹配 | `src/main/ets/repository/Repository.ets`、`rawsql/RawSqlGuards.ets` |
| 事务写入路径 | 事务内混用外层 repo/new repo 写入 | 统一复用 `txRepo` | `src/main/ets/repository/RepositoryTransactionGuardSupport.ets` |

## 3. 旧新写法对照

### 3.1 初始化：不要依赖默认 `encrypt`

```arkts
// 旧风险写法：依赖默认 encrypt
const config = new DatabaseConfig('app.db')
await OCORMInit(context, { config })
```

```arkts
// 新写法：显式声明行为
const config = new DatabaseConfig('app.db', undefined, true)
await OCORMInit(context, { config })
```

### 3.2 聚合查询：实体模式与聚合模式分离

```arkts
// 旧风险写法：实体模式混用聚合 DSL
const rows = await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['status', 'COUNT(*) AS total'])
    .groupBy('status')
    .having('COUNT(*) > ?', [1])
).get()
```

```arkts
// 新写法：原生结果模式
const rows = await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['status', 'COUNT(*) AS total'])
    .groupBy('status')
    .having('COUNT(*) > ?', [1])
).getRaw()
```

### 3.3 事务：删掉 `serializable`/`REPEATABLE_READ` 依赖

```arkts
// 旧风险写法
const options = TransactionOptions.serializable()
await repo.transactionWithOptions(async (txRepo) => {
  await txRepo.save(user)
}, options)
```

```arkts
// 新写法
const options = TransactionOptions.fromConfig({
  isolation: IsolationLevel.READ_COMMITTED,
  timeout: 10000,
  retries: 1,
  retryDelay: 200,
  readOnly: false
})

await repo.transactionWithOptions(async (txRepo) => {
  await txRepo.save(user)
}, options)
```

### 3.4 原生 SQL：全量参数化

```arkts
// 旧风险写法
await repo.rawQuery('SELECT * FROM user')
await repo.rawExecute(`UPDATE user SET name = 'Alice' WHERE id = 1`)
```

```arkts
// 新写法
await repo.rawQuery('SELECT * FROM user WHERE id = ?', [1])
await repo.rawExecute('UPDATE user SET name = ? WHERE id = ?', ['Alice', 1])
```

### 3.5 事务上下文：只用 `txRepo` 写入

```arkts
// 旧风险写法：事务中回写外层 repo
await repo.transaction(async (txRepo) => {
  await repo.save(user)
})
```

```arkts
// 新写法：事务内统一使用 txRepo
await repo.transaction(async (txRepo) => {
  await txRepo.save(user)
})
```

## 4. 批量改造清单（可直接执行）

### 4.1 扫描高风险调用点

```bash
rg -n "new DatabaseConfig\(|selectRaw\(|groupBy\(|having\(|rawQuery\(|rawQuerySafe\(|rawExecute\(|TransactionOptions\.serializable\(|REPEATABLE_READ|SERIALIZABLE|readOnly" src test
```

### 4.2 扫描事务误用（事务内写外层 repo）

```bash
rg -n "transaction\(async \(txRepo\).*repo\." src test
```

### 4.3 扫描测试桥接旧入口（历史兼容点）

```bash
rg -n "getTestState|setTestState|applyRuntimeConfigForTest|resetInstance\(" src test
```

# 02-分页与排序列表页

> 状态：已完成  
> 适用版本：ocorm 3.x（当前仓库）
> 最后更新：2026-03-28

## 场景问题
列表页通常同时要做三件事：
1. 支持按字段排序（如 `id`、`createdAt`）
2. 支持分页（页码 + 每页条数）
3. 防止前端把非法参数直接透传到查询层

如果不做参数治理，`orderBy` 会因为非法列名或方向报错，`limit/offset` 也会因为负数参数直接失败。

## 完整方案
用 `Repository.createQueryBuilder()` 组合查询，排序方向严格收敛到 `ASC | DESC`，分页统一走 `paginate`，让页码和大小自动兜底。

```arkts
import { QueryExecutor, Repository, SortDirection } from 'ocorm'

interface UserListRequest {
  page: number
  pageSize: number
  sortBy: string
  sortDir: string
}

const SORTABLE_COLUMNS: Array<string> = ['id', 'createdAt', 'name']

function normalizeSortBy(input: string): string {
  if (SORTABLE_COLUMNS.includes(input)) {
    return input
  }
  return 'id'
}

function normalizeSortDirection(input: string): SortDirection {
  const normalized = input.trim().toUpperCase()
  return normalized === 'DESC' ? 'DESC' : 'ASC'
}

export async function queryUserList(repo: Repository, request: UserListRequest) {
  const qb = repo.createQueryBuilder()
    .orderBy(normalizeSortBy(request.sortBy), normalizeSortDirection(request.sortDir))
    .paginate(request.page, request.pageSize)

  return await new QueryExecutor(qb).getPaginated()
}
```

如果你要做“下一页加载”而不是标准分页，也可以用 `limit/offset`：

```arkts
import { QueryExecutor, Repository } from 'ocorm'

export async function queryUserSlice(repo: Repository, offset: number, size: number) {
  const qb = repo.createQueryBuilder()
    .orderBy('id', 'ASC')
    .limit(size)
    .offset(offset)

  return await new QueryExecutor(qb).get()
}
```

## 验证
最小验证点：
1. `paginate` 的兜底行为是否符合预期（非法页码会被修正）
2. `getPaginated()` 返回结构是否含 `data/total/page/pageSize/totalPages`
3. 非法排序方向是否按源码抛 `InvalidConditionError`

```arkts
import { InvalidConditionError, QueryExecutor, Repository } from 'ocorm'

async function verify(repo: Repository): Promise<void> {
  const normalizedQb = repo.createQueryBuilder().paginate(-5, 0)
  console.info(normalizedQb.getPageNumber())   // 1
  console.info(normalizedQb.getPageSizeValue()) // 20

  const pageResult = await new QueryExecutor(
    repo.createQueryBuilder().orderBy('id', 'ASC').paginate(1, 10)
  ).getPaginated()

  console.info(pageResult.page, pageResult.pageSize, pageResult.totalPages)

  try {
    repo.createQueryBuilder().orderBy('id', 'UPWARD' as never)
  } catch (error) {
    if (error instanceof InvalidConditionError) {
      console.error('排序方向校验已生效:', error.message)
    }
  }
}
```

## 失败修复
1. 失败：`InvalidConditionError('ORDER BY', '非法排序方向: xxx')`  
修复：仅允许前端传 `ASC/DESC`，其它值在服务层归一化。

2. 失败：`InvalidConditionError`（排序列不存在）  
修复：对 `sortBy` 做白名单映射，不要直接信任外部字符串。

3. 失败：`LIMIT/OFFSET 值不能为负数`  
修复：标准分页场景优先 `paginate(page, size)`；游标场景先手动 `Math.max(0, value)` 再传 `limit/offset`。

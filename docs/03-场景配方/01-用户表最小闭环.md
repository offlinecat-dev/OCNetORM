# 01-用户表最小闭环

> 状态：已完成  
> 适用版本：ocorm 3.x（当前仓库）
> 最后更新：2026-03-28

## 场景问题
你要快速打通一个实体（以 `User` 为例）的最小读写闭环：
1. 新增或更新
2. 按主键读取和全量读取
3. 删除与恢复（软删除场景）
4. 识别事务内外失败语义差异

## 完整方案
用 `Repository` 直接完成 CRUD，写入结果一律检查 `success`，事务内只使用 `txRepo`。

```arkts
import { EntityData, Repository } from 'ocorm'

function makeUser(id: number, name: string): EntityData {
  const user = new EntityData('User')
  user.addProperty('id', id, 'number')
  user.addProperty('name', name, 'string')
  return user
}

export async function userCrudFlow(): Promise<void> {
  const repo = new Repository('User')

  const saveResult = await repo.save(makeUser(1, 'Alice'))
  if (!saveResult.success) {
    throw new Error(`save 失败: ${saveResult.errorMessage}`)
  }

  const one = await repo.findById(1)
  const all = await repo.findAll()
  console.info(one !== null, all.length)

  const removeResult = await repo.removeById(1)
  if (!removeResult.success) {
    throw new Error(`removeById 失败: ${removeResult.errorMessage}`)
  }
}
```

软删除实体要恢复数据时，查询要显式 `includeDeleted = true`：

```arkts
import { Repository } from 'ocorm'

async function restoreSoftDeletedPost(): Promise<void> {
  const postRepo = new Repository('Post')

  await postRepo.removeById(7)
  const hidden = await postRepo.findById(7)      // 默认查不到软删除
  const deleted = await postRepo.findById(7, true)
  console.info(hidden === null, deleted !== null)

  const restoreResult = await postRepo.restore(7)
  if (!restoreResult.success) {
    throw new Error(`restore 失败: ${restoreResult.errorMessage}`)
  }
}
```

## 验证
最小验证点：
1. `save` 失败在普通上下文返回失败结果对象
2. 事务上下文里失败会升级为异常并触发回滚
3. `saveAll` 是顺序保存，不是自动原子批处理

```arkts
import { EntityData, ExecutionError, Repository } from 'ocorm'

function makeUser(id: number, name: string): EntityData {
  const user = new EntityData('User')
  user.addProperty('id', id, 'number')
  user.addProperty('name', name, 'string')
  return user
}

async function verify(repo: Repository): Promise<void> {
  const list = await repo.saveAll([
    makeUser(11, 'A'),
    makeUser(12, 'B')
  ])
  console.info(list.length)

  try {
    await repo.transaction(async (txRepo) => {
      await txRepo.save(makeUser(21, 'first'))
      await txRepo.save(makeUser(21, 'duplicate'))
    })
  } catch (error) {
    if (error instanceof ExecutionError || error instanceof Error) {
      console.error('严格事务失败后已回滚:', error)
    }
  }
}
```

## 失败修复
1. 失败：把 `saveAll` 当成“全成全败”  
修复：需要原子性时显式包进 `transaction` 或改用 `batchInsert`。

2. 失败：事务回调里混用外部 `repo` 写入  
修复：事务回调内只使用 `txRepo`，否则会被写入守卫拦截。

3. 失败：软删除后仍用默认查询找不到数据  
修复：排查时使用 `findById(id, true)`，恢复后再走默认查询路径。

4. 失败：未检查 `SaveResult/DeleteResult.success`  
修复：所有写操作统一检查 `success` 与 `errorMessage`。

# 03-事务内批量写入

> 状态：已完成  
> 适用版本：ocorm 3.x（当前仓库）
> 最后更新：2026-03-28

## 场景问题
你需要批量写入大量数据，但同时要保证：
1. 失败可回滚（原子性）
2. 吞吐可接受
3. 调用方能拿到清晰的成功/失败明细

本仓库里 `saveAll` 与 `batchInsert` 语义不同，误用会直接导致“以为原子，实际部分成功”的事故。

## 完整方案
- 高吞吐导入：优先 `batchInsert`
- 需要事务原子性：放进 `transaction`，并且只用回调里的 `txRepo`
- 调用后必须检查 `insertedCount/totalCount/failedIndexes`

```arkts
import { BatchInsertOptions, EntityData, Repository } from 'ocorm'

function makeUser(id: number, name: string, email: string): EntityData {
  const user = new EntityData('User')
  user.addProperty('id', id, 'number')
  user.addProperty('name', name, 'string')
  user.addProperty('email', email, 'string')
  return user
}

export async function importUsersAtomically(repo: Repository): Promise<void> {
  const entities: Array<EntityData> = [
    makeUser(1001, 'Alice', 'alice@example.com'),
    makeUser(1002, 'Bob', 'bob@example.com')
  ]

  await repo.transaction(async (txRepo) => {
    const result = await txRepo.batchInsert(entities, BatchInsertOptions.createSafe())

    // 严格事务下失败通常会直接抛 ExecutionError；这里保留兜底校验
    if (!result.success || result.insertedCount !== result.totalCount) {
      throw new Error(`批量写入未全量成功: ${result.errorMessage}`)
    }
  })
}
```

如果你明确要复用 `save` 的逐条语义（而不是追求吞吐），再使用 `saveAll`：

```arkts
import { EntityData, Repository } from 'ocorm'

export async function saveUsersOneByOne(repo: Repository, users: Array<EntityData>): Promise<void> {
  const results = await repo.saveAll(users)
  for (let i = 0; i < results.length; i++) {
    if (!results[i].success) {
      console.error(`第 ${i} 项失败: ${results[i].errorMessage}`)
    }
  }
}
```

## 验证
重点验证四件事：
1. 空数组写入是成功空操作
2. `createFast()` 会关闭事务/钩子/验证
3. `BatchInsertResult` 不能只看 `success`
4. 事务回调内失败会触发回滚路径

```arkts
import { BatchInsertOptions, Repository } from 'ocorm'

async function verify(repo: Repository): Promise<void> {
  const empty = await repo.batchInsert([])
  console.info(empty.success, empty.insertedCount, empty.totalCount) // true, 0, 0

  const fast = BatchInsertOptions.createFast()
  console.info(fast.useTransaction, fast.executeHooks, fast.executeValidation) // false,false,false

  const result = await repo.batchInsert([], BatchInsertOptions.createDefault())
  console.info(result.success, result.insertedCount, result.totalCount, result.failedIndexes.length)
}
```

## 失败修复
1. 失败：把 `saveAll` 当成高性能批量导入  
修复：改用 `batchInsert`；`saveAll` 本质是顺序调用 `save`。

2. 失败：把 `BatchInsertOptions.createFast()` 当成“仅关闭事务”  
修复：明确它会同时关闭事务、钩子、验证；生产默认用 `createSafe()` 或 `createDefault()`。

3. 失败：只看 `result.success`，忽略计数和索引  
修复：统一检查 `insertedCount/totalCount/failedIndexes`，不要丢失部分成功信息。

4. 失败：事务里混用外部 `repo` 写入  
修复：事务回调内只使用 `txRepo`，避免被写入守卫拦截。

# 测试钩子与测试环境隔离

## 目标
规范 `DatabaseManagerTestHooks` 的注入与回滚流程，防止单例状态在 suite 间串扰。

## 可用测试 API（真实签名）
- `DatabaseManagerTestHooks.getState(manager?)`
- `DatabaseManagerTestHooks.setState(manager, state)`
- `DatabaseManagerTestHooks.applyRuntimeConfig(manager, config)`
- `DatabaseManagerTestHooks.resetInstance()`

## 测试注入示例：保存-注入-恢复
```ts
import { describe, it, expect } from '@ohos/hypium'
import { DatabaseConfig } from '../../main/ets/database/DatabaseConfig'
import { DatabaseManager } from '../../main/ets/database/DatabaseManager'
import { DatabaseManagerTestHooks } from '../../main/ets/database/testing/DatabaseManagerTestHooks'

describe('DatabaseManagerTestHooks isolate demo', () => {
  it('should restore runtime state', () => {
    const manager = DatabaseManager.getInstance()
    const snapshot = DatabaseManagerTestHooks.getState(manager)

    try {
      DatabaseManagerTestHooks.applyRuntimeConfig(
        manager,
        new DatabaseConfig('test_runtime.db')
          .setMaxConcurrentQueries(3)
          .setQueryTimeout(200)
      )

      expect(manager.getMaxConcurrentQueries()).assertEqual(3)
      expect(manager.getQueryTimeoutMs()).assertEqual(200)
    } finally {
      DatabaseManagerTestHooks.setState(manager, snapshot)
    }
  })
})
```

## 生命周期调用序列：测试套件推荐模板
```ts
import { beforeEach, afterEach } from '@ohos/hypium'
import { DatabaseManager } from '../../main/ets/database/DatabaseManager'
import { DatabaseManagerTestHooks, DatabaseManagerTestState } from '../../main/ets/database/testing/DatabaseManagerTestHooks'

let manager: DatabaseManager
let baseline: DatabaseManagerTestState

beforeEach(() => {
  manager = DatabaseManager.getInstance()
  baseline = DatabaseManagerTestHooks.getState(manager)
})

afterEach(() => {
  DatabaseManagerTestHooks.setState(manager, baseline)
  DatabaseManagerTestHooks.resetInstance()
})
```

## 故障复现脚本：不重置单例导致用例互相污染
```ts
import { DatabaseConfig } from '../../main/ets/database/DatabaseConfig'
import { DatabaseManager } from '../../main/ets/database/DatabaseManager'
import { DatabaseManagerTestHooks } from '../../main/ets/database/testing/DatabaseManagerTestHooks'

function reproduceSuitePollution(): void {
  const manager = DatabaseManager.getInstance()

  // Suite A 注入限流
  DatabaseManagerTestHooks.applyRuntimeConfig(
    manager,
    new DatabaseConfig('suite_a.db').setMaxConcurrentQueries(1)
  )

  // 如果此处漏掉 resetInstance，Suite B 仍继承 maxConcurrentQueries=1
  console.info(`polluted limit=${manager.getMaxConcurrentQueries()}`)
}
```

## 排障清单
- 只在测试工程引用 `database/testing/DatabaseManagerTestHooks`。
- 所有注入都要有回滚（`setState`）或重建（`resetInstance`）。
- 遇到 `RdbStore already closed`，优先检查是否回放了过期 `state.store`。

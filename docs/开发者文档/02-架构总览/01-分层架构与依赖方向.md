# 分层架构与依赖方向（架构+代码走读）

## 1. 分层决议（基于现有实现）
OCORM 当前是“模块化分层 + 受控跨层能力”而不是纯理论洋葱架构。读路径由 `query` 统一执行，写路径与事务由 `repository` 统一收口，连接与资源治理由 `database` 托底。核心依赖方向是：上层 API -> 领域执行层 -> 基础设施层，禁止业务层直接触达底层 `RdbStore`。

```text
entry/业务调用
  -> repository/Repository.ets (createQueryBuilder/find/save/transaction)
  -> query/QueryBuilder.ets (DSL 条件构建)
  -> query/QueryExecutor.ets (toSQL/get/getRaw/aggregate)
  -> query/RelationLoader.ets (关系二段加载)
  -> database/DatabaseManager.ets (withStoreLease/acquireQuerySlot/releaseQuerySlot)
  -> relationalStore.RdbStore (query/querySql/executeSql)
```

## 2. 依赖方向走读（真实入口）

```ts
// repository/Repository.ets
createQueryBuilder(): QueryBuilder {
  return new QueryBuilder(this.entityName)
}

async findAllAsync(includeDeleted: boolean = false): Promise<Array<EntityData>> {
  return await this.withSessionStore(async (repo) => {
    const queryBuilder = repo.createQueryBuilder()
    if (includeDeleted) {
      queryBuilder.withDeleted()
    }
    const executor = new QueryExecutor(queryBuilder)
    return await executor.getAsync()
  })
}
```

该入口体现了固定依赖方向：`Repository -> QueryBuilder -> QueryExecutor`。`Repository` 不直接拼接 SQL，也不接管连接池；它只负责场景入口、守卫和会话边界。

```ts
// query/QueryExecutor.ets
private async withStoreLease<T>(operation: () => Promise<T>): Promise<T> {
  const manager = DatabaseManager.getInstance()
  return await manager.withStoreLease(async (lease) => {
    this.leasedStore = lease.store
    return await operation()
  })
}

async get(): Promise<Array<EntityData>> {
  return await this.withStoreLease(async () => {
    this.ensureEntityQueryMode('SELECT')
    await DatabaseManager.getInstance().acquireQuerySlot()
    try {
      // ... query + transform + relation
      return []
    } finally {
      DatabaseManager.getInstance().releaseQuerySlot()
    }
  })
}
```

`QueryExecutor` 只消费 `DatabaseManager` 暴露的租约与槽位接口，不自己创建或销毁连接，这就是运行时依赖单向性的关键。

## 3. 请求到 SQL 的路径样例

```ts
// 业务请求示例（上层）
import { Repository, QueryExecutor, ConditionOperator } from 'ocorm'

const repo = new Repository('User')
const qb = repo
  .createQueryBuilder()
  .where('status', ConditionOperator.EQUAL, 'ACTIVE')
  .orderBy('id', 'DESC')
  .limit(20)
const result = await new QueryExecutor(qb).get()
```

```text
请求 -> Repository.createQueryBuilder()
    -> QueryBuilder.where/orderBy/limit
    -> QueryExecutor.get()
    -> QueryExecutor.buildPredicatesOrThrow(...)
    -> QueryExecutor.queryResultSet(...)
    -> RdbStore.query(...)
    -> QueryResultTransformer/DataMapper
    -> RelationLoader.loadRelation(s)
    -> 返回 EntityData[]
```

## 4. 事务生命周期样例（分层责任）

```ts
// repository/Repository.ets
async transactionWithOptions(callback: TransactionCallback, options: TransactionOptions): Promise<TransactionResult> {
  this.ensureNoNestedTransaction('transactionWithOptions')
  return await this.transactionManager.transactionWithOptions(async (store) => {
    const scopedRepository = this.createTransactionScopedRepository(store)
    await callback(scopedRepository)
  }, options)
}
```

```ts
// repository/TransactionManager.ets
async transaction(action: TransactionAction): Promise<TransactionResult> {
  const lease = await this.acquireTransactionLeaseOrThrow()
  const releaseLock = await this.acquireTransactionLock(lease.store)
  try {
    store.beginTransaction()
    await action(store)
    store.commit()
  } catch (e) {
    store.rollBack()
    throw e
  } finally {
    releaseLock()
    lease.release()
  }
}
```

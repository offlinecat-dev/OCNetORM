# 05-性能基准口径与复现

> 状态：已完成
> 适用范围：仓库内可复用的性能观测与复现实验
> 最后更新：2026-03-28

## 1. 目标
- 统一当前仓库里的性能口径。
- 明确哪些 Suite/入口能用于复现实验。
- 避免把一次随手跑出来的数字写成结论。

## 2. 推荐入口
当前仓库里和性能最相关的真实入口有：
- `entry/src/main/ets/suites/PerformanceSuite.ets`
- `entry/src/main/ets/suites/EntryOrmPerformanceBatchSuite.ets`
- `entry/src/main/ets/suites/EntryConnectionPoolSuite.ets`
- `entry/src/main/ets/testcore/CommandSuiteRunner.ets`

```arkts
import { CommandSuiteRunner } from 'src/main/ets/testcore/CommandSuiteRunner'

const result = await CommandSuiteRunner.run(context, {
  profile: 'ci-performance',
  includePerformance: true,
  suiteFilter: ['PerformanceSuite', 'EntryOrmPerformanceBatchSuite']
})
```

## 3. 复现原则
- 固定 profile
- 固定 suiteFilter
- 固定数据库配置
- 固定数据规模

```text
至少固定 4 件事:
1. profile = ci / ci-performance
2. includePerformance = true/false
3. DatabaseConfig 中的连接池 / cache / relationInClauseLimit
4. 样本数量与实体结构
```

## 4. 误用示例
下面这种“混跑所有 suite 再只截一个 duration”没有参考价值。

```arkts
await CommandSuiteRunner.run(context, {
  profile: 'default'
})
```

另一个误用是连 `PerformanceSuite` 是否启用都没固定下来。当前仓库里它默认不是永远开启。

## 5. 验收建议
- 同一条基准至少跑 3 次，记录最大/最小/中位
- 不要把一次带超时或后台清理的异常运行混进正常样本
- 需要横向比较时，保持相同 suiteFilter 和数据规模

```bash
rg -n "ci-performance|includePerformance|PerformanceSuite|EntryOrmPerformanceBatchSuite" entry/src/main/ets/suites/index.ets entry/src/main/ets/testcore/CommandSuiteRunner.ets
```

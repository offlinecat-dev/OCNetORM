# 04-回归清单-事务-查询-安全

> 状态：已完成
> 适用范围：事务、查询、安全相关回归
> 最后更新：2026-03-28

## 1. 目标
- 给当前仓库提供一份可执行的最小回归清单。
- 把事务、查询、安全三条高风险线明确挂到真实 suite/test。

## 2. 事务回归

```text
事务线最小清单:
1. 同 Repository 嵌套事务应拒绝
2. 跨 Repository 嵌套事务应拒绝
3. 未绑定事务上下文写入应阻断
4. timeout 后应进入后台清理
5. cleanup 未完成前应继续视为 active
```

对应真实入口优先看：
- `EntryOrmSecurityFixSuite`
- `EntryOrmCoreFixSuite`
- `EntryOrmConcurrencyFixSuite`
- `EntryOrmReviewReport19FixSuite`

## 3. 查询回归

```text
查询线最小清单:
1. 实体模式拒绝 selectRaw/groupBy/having
2. getRaw 返回原生结果语义
3. rawQuery/rawQuerySafe 必须参数化
4. 查询超时后迟到 ResultSet 应关闭
5. 关系加载按 relationInClauseLimit 分批
```

对应真实入口优先看：
- `EntryOrmReviewReport20FixSuite`
- `EntryOrmCoreFixSuite`
- `EntryOrmPerformanceBatchSuite`
- `EntryRelationTimeoutSuite`
- `EntryOrmObservabilityFixSuite`

## 4. 安全回归

```text
安全线最小清单:
1. raw SQL 注释/多语句/危险关键字拦截
2. rawQuery 只读约束
3. rawExecute 写路径守卫
4. 只读事务 query_only 语义
5. READ_UNCOMMITTED / READ_COMMITTED 能力边界
```

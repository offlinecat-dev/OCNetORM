## 5. 最小可跑片段（按任务路径落地）

```arkts
// 正确示例：聚合走原生模式（getRaw / aggregate）
import { QueryBuilder, QueryExecutor, ConditionOperator } from 'ocorm'

const qb = new QueryBuilder('Order')
  .where('status', ConditionOperator.EQUAL, 'PAID')
  .selectRaw(['status', 'COUNT(*) AS total'])
  .groupBy('status')
  .having('COUNT(*) > ?', [1])

const rows = await new QueryExecutor(qb).getRaw()
```

```arkts
// 常见错误示例：在实体模式入口混用 selectRaw/groupBy/having
import { QueryBuilder, QueryExecutor } from 'ocorm'

const qb = new QueryBuilder('Order')
  .selectRaw(['COUNT(*) AS total'])
  .groupBy('status')

await new QueryExecutor(qb).get()
```

预期行为：
- 正确示例命中 `QueryExecutor.getRaw` 原生模式，允许 `selectRaw/groupBy/having`。
- 错误示例会被 `QueryExecutor.ensureEntityQueryMode` 拒绝。

## 6. 故障定位脚本（快速代码路径）

```bash
# 正确示例：按“症状 -> 守卫/执行器 -> 文档章节”追溯
rg "ensureEntityQueryMode|validateAndNormalizeHavingClause|validateAndNormalizeSelectRawExpressions" OCORM/src/main/ets/query
rg "transactionWithOptions|applyIsolationLevel|applyReadOnlyMode" OCORM/src/main/ets/repository
```

```bash
# 常见错误示例：只搜测试日志，不搜守卫实现
rg "失败|异常|error" entry/src/main/ets
```

预期行为：
- 正确脚本会直接定位 fail-fast 入口函数，能快速映射到阅读路径中的对应章节。
- 错误脚本只会看到症状，不会看到行为边界。

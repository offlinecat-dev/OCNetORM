# 00-查询体系总览

> 适用版本：ocorm 3.0.2

查询体系由 `QueryBuilder`（描述查询）+ `QueryExecutor`（执行查询）构成。实体模式与原生模式必须分开使用。关系加载、缓存、守卫都在执行链路里可观测。

## 最小可执行链路
```ts
const qb = repo.createQueryBuilder()
  .where('status', ConditionOperator.EQUAL, 'active')
  .orderBy('createdAt', 'DESC')
  .paginate(1, 20)
  .with('posts')

const rows = await new QueryExecutor(qb).get()
```

```ts
const raw = await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['status', 'COUNT(*) AS total'])
    .groupBy('status')
    .having('COUNT(*) > ?', [10])
).getRaw()
```

## 误用示例（实体模式混用聚合）
```ts
await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['COUNT(*) AS total'])
).get()
```

语义结果：抛 `ExecutionError`，消息为 `实体查询不支持 selectRaw/groupBy/having，请使用 getRaw() 或 aggregate()`。

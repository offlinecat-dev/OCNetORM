# 第一次事务

## 前置条件
- `User` 实体已注册，数据库已完成初始化。
- 写操作必须通过同一个事务回调中的 `txRepo` 执行。

## 完整代码
```ts
import {
  EntityData,
  EntityDataInput,
  Repository,
  TransactionOptions,
  TransactionRollbackError
} from 'ocorm'

function makeUserEntity(name: string, age: number): EntityData {
  const input = EntityDataInput.create()
  input.set('userName', name)
  input.set('age', age)
  return EntityData.from('User', input)
}

export async function runFirstTransaction(): Promise<void> {
  const repo = new Repository('User')

  const txResult = await repo.transaction(async (txRepo) => {
    const r1 = await txRepo.save(makeUserEntity('tx-neo', 30))
    if (!r1.success) {
      throw new Error(r1.errorMessage)
    }
    const r2 = await txRepo.save(makeUserEntity('tx-trinity', 29))
    if (!r2.success) {
      throw new Error(r2.errorMessage)
    }
  })

  if (!txResult.success) {
    throw new Error(`事务执行失败: ${txResult.errorMessage}`)
  }
}

export async function runTransactionWithTimeout(): Promise<void> {
  const repo = new Repository('User')
  const options = TransactionOptions.withTimeout(1500)

  try {
    await repo.transactionWithOptions(async (txRepo) => {
      const result = await txRepo.save(makeUserEntity('tx-timeout-demo', 31))
      if (!result.success) {
        throw new Error(result.errorMessage)
      }
    }, options)
  } catch (error) {
    if (error instanceof TransactionRollbackError) {
      throw new Error(`事务回滚: ${error.message}`)
    }
    throw error
  }
}
```

## 验证步骤
1. 调用 `await runFirstTransaction()`，应成功提交两条数据。
2. 调用 `await runTransactionWithTimeout()`，在超时/失败时应抛出 `TransactionRollbackError`。
3. 事务回调内任一步骤主动抛错，最终应整体回滚。

## 常见失败
- `TransactionRollbackError`：事务内抛异常、超时、或回滚失败。
- `TransactionRollbackError`（嵌套提示）：同一 `Repository` 内嵌套调用 `transaction(...)` / `transactionWithOptions(...)`。
- `ExecutionError`：事务中的 SQL 执行失败（字段不匹配、连接异常等）。

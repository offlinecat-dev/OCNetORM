# 环境要求

## 1. 工程基线（来自真实配置）
- 根工程声明模块：`entry` + `ocorm`（`build-profile.json5` -> `modules`）
- 根工程 SDK：`targetSdkVersion: 6.0.1(21)`，`compatibleSdkVersion: 5.0.5(17)`（`build-profile.json5` -> `app.products[0]`）
- 本地构建入口：`tools/hvigor-local.cmd`（固定走 DevEco 自带 Node + hvigor）

```json5
// build-profile.json5（根）关键字段
{
  "app": {
    "products": [
      {
        "name": "default",
        "targetSdkVersion": "6.0.1(21)",
        "compatibleSdkVersion": "5.0.5(17)",
        "runtimeOS": "HarmonyOS"
      }
    ],
    "buildModeSet": [{ "name": "debug" }, { "name": "release" }]
  },
  "modules": [
    { "name": "entry", "srcPath": "./entry" },
    { "name": "ocorm", "srcPath": "./OCORM" }
  ]
}
```

## 2. 构建命令序列（可直接执行）
```powershell
# 1) 安装依赖
ohpm install

# 2) 清理历史产物
tools\hvigor-local.cmd clean

# 3) 先构建 HAR，再构建 HAP
tools\hvigor-local.cmd assembleHar
tools\hvigor-local.cmd assembleHap

# 4) 跑测试
tools\hvigor-local.cmd test
```

## 3. 本地脚本前置检查
```bat
:: tools/hvigor-local.cmd 关键逻辑
set "DEVECO_HOME=D:\Program Files\Huawei\DevEco Studio"
set "NODE_EXE=%DEVECO_HOME%\tools\node\node.exe"
set "HVIGOR_JS=%DEVECO_HOME%\tools\hvigor\hvigor\bin\hvigor.js"
"%NODE_EXE%" "%HVIGOR_JS%" --no-daemon %*
```

## 4. 常见错误与修复命令

### 4.1 `node not found` / `hvigor entry not found`
```powershell
# 1) 打开脚本并修正 DEVECO_HOME
notepad tools\hvigor-local.cmd

# 2) 验证关键文件存在
Test-Path "D:\Program Files\Huawei\DevEco Studio\tools\node\node.exe"
Test-Path "D:\Program Files\Huawei\DevEco Studio\tools\hvigor\hvigor\bin\hvigor.js"

# 3) 重新执行构建
tools\hvigor-local.cmd clean
tools\hvigor-local.cmd assembleHar
```

### 4.2 `entry` 找不到本地 `ocorm`
```powershell
# 1) 确认本地文件依赖未被改坏
Get-Content entry\oh-package.json5

# 2) 重新安装依赖并重建
ohpm install
tools\hvigor-local.cmd clean
tools\hvigor-local.cmd assembleHar
tools\hvigor-local.cmd assembleHap
```

# 12-常见误用与失败语义

只列最常见且可复现的误用。

## 误用 1：实体模式混用聚合
```ts
await new QueryExecutor(
  repo.createQueryBuilder().selectRaw(['COUNT(*) AS total'])
).get()
```

语义结果：抛 `实体查询不支持 selectRaw/groupBy/having，请使用 getRaw() 或 aggregate()`。

## 误用 2：`whereExists` 关系不存在
```ts
repo.createQueryBuilder().whereExists('unknownRelation', sub => {
  sub.where('id', ConditionOperator.GREATER, 0)
})
```

语义结果：抛 `RelationNotFoundError`。

## 误用 3：`update/delete` 无 WHERE
```ts
await new QueryExecutor(repo.createQueryBuilder()).delete()
```

语义结果：抛 `ExecutionError('DELETE', 'DELETE 需要至少一个 WHERE 条件')`。

## 误用 4：`withLazy` 嵌套路径
```ts
repo.createQueryBuilder().withLazy('posts.comments')
```

语义结果：关系不存在错误（只支持一级关系）。

## 正确替代
```ts
const rows = await new QueryExecutor(
  repo.createQueryBuilder()
    .selectRaw(['status', 'COUNT(*) AS total'])
    .groupBy('status')
    .having('COUNT(*) > ?', [1])
).getRaw()
```

# 模块职责地图（架构+代码走读）

## 1. 模块职责与边界
`repository` 负责写路径和事务入口，`query` 负责读路径 DSL 与执行，`database` 负责连接生命周期与并发治理。`RelationLoader` 作为查询后处理器，只处理关联加载，不处理事务状态。边界约束是：UI/业务层不得直接依赖 `database` 的具体连接能力。

```text
repository/Repository.ets
  - save/remove/batchInsert/rawExecute
  - transaction/transactionWithOptions
query/QueryBuilder.ets
  - where/groupBy/having/selectRaw
query/QueryExecutor.ets
  - get/getOne/getRaw/aggregate
query/RelationLoader.ets
  - loadRelation/loadRelationsParallel
repository/TransactionManager.ets
  - transaction/transactionWithOptions/hasAnyActiveTransaction
database/DatabaseManager.ets
  - initialize/withStoreLease/acquireQuerySlot/releaseQuerySlot/close
```

## 2. 读路径职责切面

```ts
// query/QueryBuilder.ets
where(column: string, operator: ConditionOperator, value: ValueType): QueryBuilder

groupBy(column: string): QueryBuilder
having(clause: string, args: Array<ValueType> = []): QueryBuilder
selectRaw(expressions: Array<string>): QueryBuilder
```

`QueryBuilder` 只保存查询意图，不执行数据库操作。执行语义全部下沉到 `QueryExecutor`，因此职责没有混乱。

```ts
// query/QueryExecutor.ets
private ensureEntityQueryMode(operation: string): void {
  if (this.queryBuilder.hasSelectRaw() ||
      this.queryBuilder.hasGroupBy() ||
      this.queryBuilder.getHavingClause().length > 0) {
    throw new ExecutionError(operation, '实体查询不支持 selectRaw/groupBy/having，请使用 getRaw() 或 aggregate()')
  }
}

async getRaw(): Promise<Array<ResultSetRow>> {
  return await this.withStoreLease(async () => {
    await DatabaseManager.getInstance().acquireQuerySlot()
    try {
      // RAW_SELECT 路径
      return []
    } finally {
      DatabaseManager.getInstance().releaseQuerySlot()
    }
  })
}
```

## 3. 请求到 SQL 的路径样例（模块协作）

```text
Repository.findById(id)
  -> Repository.withSessionStore(...)
  -> store.query(predicates)
  -> ResultSetUtils.toRow(...)
  -> DataMapper.fromResultSetRow(...)

Repository.createQueryBuilder() + QueryExecutor.get()
  -> QueryExecutor.toSQL()/buildPredicatesOrThrow(...)
  -> RdbStore.query(...)
  -> QueryResultTransformer
  -> RelationLoader.loadRelation/loadRelationsParallel
```

```ts
// repository/Repository.ets (findById 关键流程)
async findById(id: ValueType, includeDeleted: boolean = false): Promise<EntityData | null> {
  return await this.withSessionStore(async (repo) => {
    const store = repo.getStore()
    const pkColumn = repo.entityMetadata.getPrimaryKeyColumn()
    const predicates = new relationalStore.RdbPredicates(repo.entityMetadata.tableName)
    predicates.equalTo(pkColumn.columnName, repo.toRdbValue(id))
    const resultSet = await store.query(predicates)
    if (resultSet.goToFirstRow()) {
      const row = ResultSetUtils.toRow(resultSet, repo.entityMetadata)
      return repo.dataMapper.fromResultSetRow(row)
    }
    return null
  })
}
```

## 4. 事务生命周期样例（职责拆分）

```ts
// repository/Repository.ets
async transaction(callback: TransactionCallback): Promise<TransactionResult> {
  this.ensureNoNestedTransaction('transaction')
  return await this.transactionManager.transaction(async (store) => {
    const scopedRepository = this.createTransactionScopedRepository(store)
    await callback(scopedRepository)
  })
}
```

```text
Repository.transaction(...)
  -> TransactionManager.transaction(...)
  -> acquireTransactionLeaseOrThrow()
  -> acquireTransactionLock(store)
  -> beginTransaction
  -> action(store)
  -> commit / rollBack
  -> releaseLock + lease.release
```

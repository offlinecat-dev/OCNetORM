# 实体模型与元数据

## 1. 这一层解决什么问题
OCORM 运行时不直接依赖实体类反射，而是依赖一套显式元数据（实体、列、关系、索引、作用域）。
查询、Schema 构建、关系加载、仓储写入都会先读取这套元数据。
因此“实体建模正确”本质上是“元数据注册完整且时序正确”。

## 2. 核心对象（真实 API）

### 2.1 EntityMetadata（实体级元数据）
对应源码：`src/main/ets/core/EntityMetadata.ets`

关键字段：
- `entityName`：实体名
- `tableName`：表名（默认等于实体名）
- `columns`：`Array<ColumnMetadata>`
- `indexes`：`Array<IndexMetadata>`
- `softDelete` / `deletedAtColumn`

关键行为：
- `addColumn(column)`：直接追加，不做去重。
- `addIndex(index)`：按“索引名或签名（unique + 列组合）”去重。
- `getColumnByProperty/getColumnByName/getPrimaryKeyColumn`：查询映射入口。

### 2.2 ColumnMetadata（列元数据）
对应源码：`src/main/ets/core/ColumnMetadata.ets`

关键字段：
- `propertyName`（ArkTS 属性名）
- `columnName`（数据库列名）
- `columnType`、`isPrimaryKey`、`isAutoIncrement`、`isNullable`、`isUnique`、`defaultValue`、`length`

### 2.3 RelationMetadata / ManyToManyMetadata / MorphToMetadata
对应源码：
- `src/main/ets/core/RelationMetadata.ets`
- `src/main/ets/core/ManyToManyMetadata.ets`
- `src/main/ets/core/MorphToMetadata.ets`

关系查询路径、关系加载器都以这些对象为输入，不读取装饰器语法本身。

### 2.4 MetadataStorage（全局元数据中心）
对应源码：`src/main/ets/core/MetadataStorage.ets`

单例：`MetadataStorage.getInstance()`

核心职责：
- 注册：`registerEntity/registerColumn/registerRelation/registerManyToMany/registerMorphTo`
- 查询：`getEntityMetadata/getAllEntities/getRelation/getRelations/getManyToManyRelation/getMorphToRelation`
- 生命周期：`clear()`、`resetInstance()`（测试隔离）

## 3. 推荐注册时机与顺序（必须）
建议在应用初始化阶段一次性注册完成：
1. `registerEntity` / `defineEntity`
2. 注册列（普通列、主键、软删列）
3. 注册关系（Relation / ManyToMany / MorphTo）
4. 初始化数据库与仓储

原因：
- `QueryBuilder` 构造时就会读取实体元数据；未注册会直接抛 `EntityNotRegisteredError`。
- 关系路径（如 `with('posts.comments')`）在注册阶段缺失会直接抛 `RelationNotFoundError`。

## 4. 可运行示例（直接用元数据 API）
```ts
import {
  MetadataStorage,
  ColumnMetadata,
  RelationMetadata,
  RelationType
} from 'ocorm'

const storage = MetadataStorage.getInstance()

storage.registerEntity('User', 'users')
storage.registerEntity('Post', 'posts')

const userId = new ColumnMetadata('id', 'id')
userId.setPrimaryKey(true).setAutoIncrement(true).setType(1)
storage.registerColumn('User', userId)
storage.registerColumn('User', new ColumnMetadata('name', 'name'))

const postId = new ColumnMetadata('id', 'id')
postId.setPrimaryKey(true).setAutoIncrement(true).setType(1)
storage.registerColumn('Post', postId)
storage.registerColumn('Post', new ColumnMetadata('userId', 'user_id'))

storage.registerRelation(
  'User',
  new RelationMetadata(RelationType.ONE_TO_MANY, 'User', 'Post', 'posts', 'user_id')
)
```

## 5. 边界条件与失败语义

### 5.1 实体与列
- 重复 `registerEntity('User')`：抛 `DuplicateEntityError`（来自 `MetadataStorage.registerEntity`）。
- 未注册实体就 `registerColumn/registerRelation`：抛 `EntityNotRegisteredError`。
- 直接 `registerColumn` 不去重：同一属性可被重复追加（调用方需自控）。

### 5.2 关系
- `registerRelation` 会校验 `sourceEntity`，并在 `targetEntity` 非空时校验目标实体存在。
- `registerMorphTo` 不要求预先注册目标实体（目标由 `typeMap/typeValue` 运行时解析）。

### 5.3 测试隔离
- `clear()` 清实体/关系/实例映射。
- `resetInstance()` 重建单例。
- 测试中通常 `beforeEach` 调 `MetadataStorage.resetInstance()`。

## 6. 与测试套件对齐的核对点
- 端到端（元数据注册 -> 查询）示例：
```ts
import {
  MetadataStorage,
  ColumnMetadata,
  RelationMetadata,
  RelationType,
  Repository,
  QueryExecutor
} from 'ocorm'

const storage = MetadataStorage.getInstance()
storage.registerEntity('User', 'users')
storage.registerEntity('Post', 'posts')

const userId = new ColumnMetadata('id', 'id')
userId.setPrimaryKey(true).setAutoIncrement(true)
storage.registerColumn('User', userId)
storage.registerColumn('User', new ColumnMetadata('name', 'name'))

const postId = new ColumnMetadata('id', 'id')
postId.setPrimaryKey(true).setAutoIncrement(true)
storage.registerColumn('Post', postId)
storage.registerColumn('Post', new ColumnMetadata('userId', 'user_id'))

storage.registerRelation(
  'User',
  new RelationMetadata(RelationType.ONE_TO_MANY, 'User', 'Post', 'posts', 'user_id')
)

const qb = new Repository('User').createQueryBuilder().with('posts')
const rows = await new QueryExecutor(qb).get()
```

- 错误建模示例 + 运行期后果（关系名未注册）：
```ts
import { MetadataStorage, Repository, QueryExecutor } from 'ocorm'

MetadataStorage.resetInstance()
MetadataStorage.getInstance().registerEntity('User', 'users')

// 运行时后果：with('posts') 校验关系路径失败，抛 RelationNotFoundError（ORM501）
const qb = new Repository('User').createQueryBuilder().with('posts')
await new QueryExecutor(qb).get()
```

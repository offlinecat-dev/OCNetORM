# 装饰器建模方式

## 1. 当前可用装饰器（真实导出）
OCORM 当前装饰器导出来自 `src/main/ets/decorators/index.ets`：
- 实体：`Table`
- 列：`Column`
- 主键：`PrimaryKey`
- 时间列：`CreatedAt`、`UpdatedAt`
- 软删除：`SoftDelete`
- 辅助：`inferColumnType`

注意：当前没有 `@ManyToOne/@OneToMany/@ManyToMany/@MorphTo` 这类关系装饰器导出。关系建模仍使用 `MetadataStorage.registerRelation/registerManyToMany/registerMorphTo`。

## 2. 装饰器执行模型与时序

### 2.1 关键事实
- `Column/PrimaryKey/CreatedAt/UpdatedAt/SoftDelete` 最终都调用 `registerColumn`。
- `registerColumn` 内部会先 `ensureEntityRegistered`，实体未注册直接抛 `EntityNotRegisteredError`。

### 2.2 实践建议
- 在列装饰器生效前，必须保证实体已经注册（`registerEntity` 或 `Table`）。
- 在 ArkTS 场景下，测试用例采用“显式调用装饰器函数”保证时序（可控、稳定）。

## 3. 可运行示例（推荐的可控写法）
```ts
import {
  MetadataStorage,
  registerEntity,
  Table,
  Column,
  PrimaryKey,
  CreatedAt,
  UpdatedAt,
  SoftDelete,
  ColumnType
} from 'ocorm'

MetadataStorage.resetInstance()

class UserEntity {}
const user = new UserEntity()

// 先注册实体
Table('users')(user.constructor as Function)

// 再注册列
PrimaryKey({ autoIncrement: true })(user, 'id')
Column({ name: 'user_name', type: ColumnType.TEXT, nullable: false })(user, 'name')
CreatedAt()(user, 'createdAt')
UpdatedAt()(user, 'updatedAt')
SoftDelete()(user, 'deletedAt')
```

## 4. 各装饰器默认值与限制

### 4.1 PrimaryKey
- `autoIncrement` 默认 `true`
- 默认 `INTEGER + not null`

### 4.2 CreatedAt / UpdatedAt
默认：
- 列名 `created_at` / `updated_at`
- 类型 `INTEGER`
- `nullable = false`

### 4.3 SoftDelete
默认：
- 列名 `deleted_at`
- 类型 `INTEGER`
- `nullable = true`
- 额外更新 `EntityMetadata.setSoftDelete(true, columnName)`

### 4.4 inferColumnType
映射规则：
- `string -> TEXT`
- `number -> REAL`
- `boolean -> INTEGER`
- `Date -> INTEGER`
- 其它 -> `TEXT`

## 5. 注册与覆盖行为
- `registerEntity` 对重复实体是“忽略重复而非抛错”的包装。
- 已有实体再传 `tableName`：仅当旧值仍是默认实体名时，允许更新。
- 列注册是追加语义；如需防重复，调用方自行保证。

## 6. 边界条件
- 先 `Column(...)` 后 `registerEntity(...)`：会抛 `EntityNotRegisteredError`。
- 软删除列改名（`SoftDelete({ name: 'deleted_time' })`）会同步更新实体软删列名。
- 装饰器建模与 `defineEntity` 可混用，但必须保证最终元数据不冲突。

## 7. 从装饰器建模到查询使用（实战）
```ts
import {
  MetadataStorage,
  Table,
  PrimaryKey,
  Column,
  SoftDelete,
  ColumnType,
  Repository,
  QueryExecutor
} from 'ocorm'

MetadataStorage.resetInstance()

class UserEntity {}
const user = new UserEntity()

Table('users')(user.constructor as Function)
PrimaryKey({ autoIncrement: true })(user, 'id')
Column({ name: 'name', type: ColumnType.TEXT, nullable: false })(user, 'name')
SoftDelete({ name: 'deleted_at' })(user, 'deletedAt')

const qb = new Repository('UserEntity').createQueryBuilder()
const rows = await new QueryExecutor(qb).get()
```

## 8. 错误建模示例 + 运行期后果
```ts
import { MetadataStorage, Column, ColumnType } from 'ocorm'

MetadataStorage.resetInstance()

class BrokenUser {}
const user = new BrokenUser()

// 未先执行 Table/registerEntity
Column({ name: 'name', type: ColumnType.TEXT })(user, 'name')

// 运行时后果：registerColumn 触发 ensureEntityRegistered，抛 EntityNotRegisteredError
```

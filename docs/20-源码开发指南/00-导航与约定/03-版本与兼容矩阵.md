# 版本与兼容矩阵

> 本文只给“可验证口径”：每个结论都能在仓库代码或配置中定位。

## 1. 版本口径（单一真相）

| 维度 | 当前值 | 代码来源 |
|---|---|---|
| HAR 包版本 | `3.0.2` | `OCORM/BuildProfile.ets` (`HAR_VERSION`) |
| 包管理版本 | `3.0.2` | `OCORM/oh-package.json5` (`version`) |
| 主入口文件 | `Index.ets` | `OCORM/oh-package.json5` (`main`) |

口径约定：
- 以 `BuildProfile.ets` + `oh-package.json5` 同时一致为有效版本。
- 对外 API 能否使用，以 `OCORM/Index.ets` 是否导出为准。

## 4. API 能力兼容矩阵（事务/查询）

### 4.1 事务隔离与只读

| 能力 | 状态 | 实际语义 | 代码来源 |
|---|---|---|---|
| `READ_COMMITTED` | 支持 | 默认级别；若 `PRAGMA read_uncommitted` 不支持，则按默认语义继续并记录警告。 | `TransactionManager.applyIsolationLevel` |
| `READ_UNCOMMITTED` | 条件支持 | 依赖 `PRAGMA read_uncommitted = 1`；不支持时 fail-fast。 | `TransactionManager.applyIsolationLevel` |
| `REPEATABLE_READ` | 不支持 | 执行期显式拒绝。 | `TransactionManager.applyIsolationLevel` |
| `SERIALIZABLE` | 不支持 | 创建便捷选项时直接抛错。 | `TransactionOptions.serializable` |
| `readOnly` | 条件支持 | 依赖 `PRAGMA query_only`；不支持时 fail-fast。 | `TransactionManager.applyReadOnlyMode` |

### 4.2 事务超时语义

| 能力 | 状态 | 实际语义 | 代码来源 |
|---|---|---|---|
| `transactionWithOptions(timeout)` | 支持 | 超时时调用方先返回失败，后台继续清理并在完成后释放锁/租约。 | `TransactionManager.transactionWithOptionsInternal`、`releaseResourcesWhenCleanupSettles` |

### 4.3 查询模式边界

| 模式 | 入口 | 边界 | 代码来源 |
|---|---|---|---|
| 实体模式 | `get/getOne/getAsync/count/getPaginated` | 禁止 `selectRaw/groupBy/having`。 | `QueryExecutor.ensureEntityQueryMode` |
| 原生模式 | `getRaw/aggregate` | 用于 `selectRaw/groupBy/having` 场景。 | `QueryExecutor.getRaw`、`QueryExecutor.aggregate` |
| HAVING 安全约束 | `having` | 必须参数化，且占位符数量与参数数量一致。 | `QueryBuilderGuards.validateAndNormalizeHavingClause` |
| `selectRaw` 白名单 | `selectRaw` | 仅允许标识符或聚合表达式白名单。 | `QueryBuilderGuards.validateAndNormalizeSelectRawExpressions` |

## 5. 测试档位矩阵（entry）

| 档位 | 性能套件 | 选择方式 | 代码来源 |
|---|---|---|---|
| `default` | 默认不包含 | `profile='default'` 且未强制 include | `entry/src/main/ets/suites/index.ets` |
| `ci` | 默认不包含 | `CommandSuiteRunner` 默认 profile | `entry/src/main/ets/testcore/CommandSuiteRunner.ets` |
| `ci-performance` | 包含 | profile 命中后自动启用 | `entry/src/main/ets/suites/index.ets` |
| 任意 profile + `includePerformance` | 可覆盖 | `includePerformance` 优先级高于 profile | `entry/src/main/ets/suites/index.ets` |

补充口径：
- 命令行启动参数由 `EntryAbility` 解析：`suiteProfile`、`includePerformance`、`suiteFilter`。  
  代码来源：`entry/src/main/ets/entryability/EntryAbility.ets`。
- `total` 统计来自 `TestRunner` + `TestReporter` 对已注册用例总和的汇总。  
  代码来源：`entry/src/main/ets/testcore/TestRunner.ets`、`TestReporter.ets`。
- 当前回归基线（测试仪表盘记录）为：`default/ci=413`，`ci-performance=422`；性能档位多出的用例来自 `PerformanceSuite`。

## 7. 版本一致性校验（脚本化）

```bash
# 正确示例：同一版本号在 BuildProfile 与 oh-package 中保持一致
rg "HAR_VERSION|version|main" OCORM/BuildProfile.ets OCORM/oh-package.json5
```

```bash
# 常见错误示例：只改一个文件（会造成口径漂移）
rg "HAR_VERSION" OCORM/BuildProfile.ets
```

预期行为：
- 正确校验应同时看到 `HAR_VERSION = '3.0.2'` 与 `oh-package.json5.version = "3.0.2"`，且 `main = "Index.ets"`。
- 错误流程会遗漏双源一致性检查。

## 8. 配置键最小对照（可直接复核）

```json5
// 正确示例：根工程 build-profile 关键键（节选）
{
  "app": {
    "products": [
      {
        "targetSdkVersion": "6.0.1(21)",
        "compatibleSdkVersion": "5.0.5(17)"
      }
    ],
    "buildModeSet": [
      { "name": "debug" },
      { "name": "release" }
    ]
  },
  "modules": [
    { "name": "entry", "srcPath": "./entry" },
    { "name": "ocorm", "srcPath": "./OCORM" }
  ]
}
```

```json5
// 常见错误示例：把 HAR 模块 type 改成 entry（破坏库交付语义）
{
  "module": {
    "name": "ocorm",
    "type": "entry"
  }
}
```

预期行为：
- 正确配置中 `OCORM/src/main/module.json5` 必须是 `"type": "har"`。
- 错误配置会使模块角色从库变成应用，构建与发布链路失真。

## 9. 兼容矩阵故障定位脚本

```bash
# 正确示例：一次核对三类 module.type 与 deviceTypes
rg "\"type\": \"har\"|\"type\": \"entry\"|\"type\": \"feature\"|\"deviceTypes\"" OCORM/src/main/module.json5 entry/src/main/module.json5 OCORM/src/ohosTest/module.json5
```

```bash
# 常见错误示例：只核对 entry，忽略 har/ohosTest
rg "\"module\"" entry/src/main/module.json5
```

预期行为：
- 正确脚本可同时覆盖 HAR/HAP/feature 三类模块。
- 错误脚本会漏检测试模块或库模块的兼容边界。

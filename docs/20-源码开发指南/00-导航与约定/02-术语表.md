# 术语表

> 目标：统一词义，避免文档和代码“同词异义”。

## 1. 交付与模块术语

| 术语 | 定义 | 代码来源 |
|---|---|---|
| `HAR` | OCORM 库模块交付形态，模块类型为 `har`。 | `src/main/module.json5` |
| `HAP(entry)` | 示例与集成测试宿主应用，模块类型为 `entry`。 | `../entry/src/main/module.json5` |
| `ohosTest(feature)` | OCORM 设备测试模块，类型为 `feature`。 | `src/ohosTest/module.json5` |
| `公共导出面` | 对使用方可见的稳定 API 集合，以 `Index.ets` 实际 `export` 为准。 | `Index.ets` |

## 4. 查询体系术语

| 术语 | 定义 | 代码来源 |
|---|---|---|
| `QueryBuilder` | 链式查询 DSL，负责收集条件、关系、分页、排序、selectRaw/groupBy/having 等声明。 | `src/main/ets/query/QueryBuilder.ets` |
| `QueryExecutor` | 执行 QueryBuilder 并映射结果；支持实体模式与原生模式。 | `src/main/ets/query/QueryExecutor.ets` |
| `实体模式` | `get/getOne/getAsync/count/getPaginated` 等返回实体结构；禁止 `selectRaw/groupBy/having`。 | `QueryExecutor.ensureEntityQueryMode` |
| `原生模式` | `getRaw/aggregate`，用于原生表达式与聚合场景。 | `src/main/ets/query/QueryExecutor.ets` |
| `with` | 预加载关系（支持路径语义，由 relation support 处理）。 | `QueryBuilder.with` |
| `withLazy` | 延迟关系声明（当前实现以单级关系名为核心约束）。 | `QueryBuilder.withLazy` 与关联支持实现 |
| `withCount` | 关系计数注入（带别名规则校验）。 | `QueryBuilder.withCount` |
| `whereExists` | 通过子查询过滤主查询结果。 | `QueryBuilder.whereExists` |
| `QueryCache` | 一级查询缓存，按 namespace+entity+id 组 key；TTL 过期与容量淘汰。 | `src/main/ets/query/QueryCache.ets` |
| `QueryExecutor 并发复用保护` | 同一个 QueryExecutor 实例并发执行会 fail-fast。 | `QueryExecutor.operationInProgress` |

## 5. 事务与一致性术语

| 术语 | 定义 | 代码来源 |
|---|---|---|
| `transaction` | 基础事务入口（BEGIN/COMMIT/ROLLBACK）。 | `TransactionManager.transaction` |
| `transactionWithOptions` | 带 `timeout/retries/isolation/readOnly` 的事务入口。 | `TransactionManager.transactionWithOptions` |
| `IsolationLevel.READ_COMMITTED` | 默认隔离级别；若平台不支持 `PRAGMA read_uncommitted`，按默认语义继续并记录警告。 | `TransactionManager.applyIsolationLevel` |
| `IsolationLevel.READ_UNCOMMITTED` | 依赖 `PRAGMA read_uncommitted=1`；平台不支持时 fail-fast。 | `TransactionManager.applyIsolationLevel` |
| `REPEATABLE_READ / SERIALIZABLE` | 当前不提供强语义支持；会显式拒绝。 | `TransactionOptions` 与 `TransactionManager.applyIsolationLevel` |
| `readOnly 事务` | 通过 `PRAGMA query_only` 切换；平台不支持时 fail-fast。 | `TransactionManager.applyReadOnlyMode` |
| `TransactionTimeoutPending` 语义 | 调用方先收到超时，但后台继续清理事务与释放资源。 | `TransactionManager.transactionWithOptionsInternal` |
| `active/pending cleanup 事务状态` | 事务活跃统计包含正在清理的 pending 状态。 | `TransactionManager.hasAnyActiveTransaction` |

## 6. 安全守卫术语

| 术语 | 定义 | 代码来源 |
|---|---|---|
| `HAVING 参数化守卫` | HAVING 必须使用 `?` 占位，且占位符数量必须与参数数量一致。 | `QueryBuilderGuards.validateAndNormalizeHavingClause` |
| `selectRaw 白名单守卫` | 仅允许标识符或聚合表达式模式，不允许危险 token。 | `QueryBuilderGuards.validateAndNormalizeSelectRawExpressions` |
| `排序方向规范化` | `orderBy` 仅允许 `ASC/DESC`。 | `QueryBuilderGuards.normalizeSortDirectionOrThrow` |

## 8. 术语到代码：最小片段对照

```arkts
// 正确示例：TransactionOptions + IsolationLevel 的真实用法
import { TransactionOptions, IsolationLevel } from 'ocorm'

const options = TransactionOptions.fromConfig({
  isolation: IsolationLevel.READ_COMMITTED,
  timeout: 5000,
  retries: 1,
  retryDelay: 100,
  readOnly: true
})
```

```arkts
// 常见错误示例：把 SERIALIZABLE 当作可用能力
import { TransactionOptions } from 'ocorm'

const options = TransactionOptions.serializable()
```

预期行为：
- 正确示例会得到可执行配置对象。
- 错误示例会立即抛错（当前实现不支持 `SERIALIZABLE`）。

```arkts
// 正确示例：DatabaseConfig 链式配置真实键
import { DatabaseConfig } from 'ocorm'

const config = new DatabaseConfig('ocorm.db')
  .setQueryCache(true, 200, 60000)
  .setQueryTimeout(3000)
  .setMaxConcurrentQueries(8)
  .setConnectionPool({ enabled: true, minSize: 1, maxSize: 4, acquireTimeoutMs: 5000 })
  .setPooledWriteGuardMode('global')
```

```arkts
// 常见错误示例：假想的配置方法（源码中不存在）
import { DatabaseConfig } from 'ocorm'

const config = new DatabaseConfig('ocorm.db')
config.enableSqlAudit(true)
```

预期行为：
- 正确示例对应 `DatabaseConfig.ets` 已实现方法。
- 错误示例会在编译阶段报“方法不存在”。

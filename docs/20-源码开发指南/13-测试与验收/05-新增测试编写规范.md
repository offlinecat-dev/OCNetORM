# 05-新增测试编写规范

> 状态：已完成
> 适用范围：`entry/src/main/ets/testcore/*` 与 `entry/src/main/ets/suites/*`
> 最后更新：2026-03-28

## 1. 目标
- 让新增测试遵守当前仓库的真实测试框架。
- 避免“自己写一套测试运行逻辑”的分叉。

## 2. 正确写法
新的 case 应继承 `TestCase`，而不是自己随手写一个 async 函数。

```arkts
import { TestCase } from '../testcore/TestCase'
import { Assert } from '../testcore/Assertions'

class SampleCase extends TestCase {
  readonly name: string = 'SampleCase'
  readonly description: string = '示例测试'

  async run(): Promise<void> {
    Assert.equals(1 + 1, 2)
  }
}
```

新的 suite 应继承 `TestSuite`，并通过 `addCase(...)` 注册用例。

```arkts
import { TestSuite } from '../testcore/TestSuite'

class SampleSuite extends TestSuite {
  readonly name: string = 'SampleSuite'
  readonly description: string = '示例套件'

  constructor() {
    super()
    this.addCase(new SampleCase())
  }
}
```

如果要命令式运行某批 suite，应复用 `CommandSuiteRunner`。

```arkts
import { CommandSuiteRunner } from '../testcore/CommandSuiteRunner'

const result = await CommandSuiteRunner.run(context, {
  profile: 'ci',
  suiteFilter: ['SampleSuite'],
  stopOnFirstFailure: true
})
```

## 3. 断言规范
当前仓库统一使用 `Assert`，而不是随意抛裸 `Error`。

```arkts
Assert.equals(actual, expected)
Assert.isTrue(flag)
await Assert.throws(async () => {
  await repo.rawQuery('SELECT * FROM user')
}, 'ExecutionError')
```

## 4. 误用示例
下面这些都不推荐：

```arkts
// 误用 1：不用 TestCase / TestSuite，自建执行框架
await someAsyncTest()
```

```arkts
// 误用 2：直接 throw new Error 做断言
if (actual !== expected) {
  throw new Error('failed')
}
```

```arkts
// 误用 3：在 Hypium List.test.ets 里直接 new Suite 并手动跑
const suite = new SampleSuite()
```

## 5. 编写检查清单
- 是否放在正确目录层级
- 是否继承 `TestCase` / `TestSuite`
- 是否复用 `Assert`
- 是否明确命名与问题域
- 是否需要同步到 `suites/index.ets` 或 `entry/src/test/List.test.ets`

```text
新增测试顺序:
1. 先判断是库级单测还是 entry 集成回归
2. 选正确目录
3. 写 TestCase
4. 放进 TestSuite
5. 再决定是否挂入口
```
